Notes

Geography

Detecting geography requires us to spot column headings in your CSV file. That means the columns have to be things that we expect. For latitude/longitude (WGS84) you'll need two columns titled Latitude/Longitude, lat/lon or geox/geoy. For OS Grid References you should have columns titled Easting/Northing. For LSOA-based data you'll need a column titled LSOA11CD or LSOA (interpreted as 2011 codes) and that column will need to contain valid ONS LSOA codes e.g. E01012334.

WGS84 and OSGB86 will result in a GeoJSON file containing points. LSOA2011-based data will result in polygons derived from the ONS Lower Layer Super Output Areas (December 2011) Boundaries EW BGC (OGL). Other columns in the CSV file will be added as properties to each feature in the GeoJSON.

Privacy

If your file contains coordinates (WGC84 or OSGB86) no data is sent from this page. If your file is LSOA-based, this page will have to make a series of requests to the server for the polygons for each LSOA in your dataset. This means the server gets to know which LSOAs are in your data. Please don't add LSOA-based data if this is a problem. If your CSV file contains a huge number of LSOAs your browser may struggle.
